# 语雀 Obsidian 导入导出

Chrome/Edge Manifest V3 扩展，用于在语雀与本地 Obsidian Vault 之间手动导入、导出 Markdown。

## 功能

- 语雀文档页：将当前文档导出到 Obsidian，同路径文件直接覆盖。
- 语雀知识库页：从同名 Obsidian 文件夹导入单篇 Markdown，或导入整个知识库。
- Obsidian 导入语雀时，本地标题、目录和正文直接覆盖匹配的语雀文档。
- 整库导入只创建或覆盖本地存在的文档，不删除语雀中的其他文档。
- 保留知识库和目录层级；导出时下载正文图片，导入时上传本地图片和附件。

## 安装

1. 在 Obsidian 中安装并启用 **Local REST API with MCP** 插件，确认 HTTP 服务地址和 API Key。
2. 打开 `chrome://extensions/`，启用开发者模式。
3. 点击“加载已解压的扩展程序”，选择本目录。
4. 点击扩展图标，填写 `http://127.0.0.1:27123` 和 API Key，先测试连接。
5. 登录语雀后，在文档页使用“导出到 Obsidian”，或在知识库页使用“从 Obsidian 导入”。

扩展不保存语雀 Cookie，也不需要 Personal Access Token；语雀写入使用当前页面的登录态和编辑权限。

## 目录规则

语雀知识库 `AI智能体` 对应：

```text
Vault/
└─ AI智能体/
   ├─ _assets/
   └─ ChatGPT/
      └─ 教程.md
```

语雀文档导出后只会在 YAML 属性中写入一条可直接打开的 `语雀链接`。再次导入时，扩展会从链接中识别原语雀文档；旧版 `yuque_source` 和 `yuque_sync` 属性仍可兼容读取，并会在下次导出时自动替换成链接。

## 导入规则

- 旧版笔记优先按当前知识库的 `doc_id` 匹配已有语雀文档。
- 新版笔记优先按 `语雀链接` 中的文档 slug 匹配。
- 没有有效来源标识时，按“相对目录 + 标题”匹配。
- 匹配到一篇时直接覆盖标题、位置和正文；没有匹配时新建。
- 同一目录存在多个同名文档时，该篇停止并显示错误。
- 整库导入逐篇执行；普通单篇失败不会影响后续文件，登录或权限失效会停止整批任务。

## 注意

语雀页面接口是网页当前使用的内部接口，不是稳定的公开 API。扩展会在登录过期、权限不足或接口返回结构变化时停止写入。首版主要处理普通文档和 Markdown，不处理表格、画板、思维导图等特殊文档类型。
